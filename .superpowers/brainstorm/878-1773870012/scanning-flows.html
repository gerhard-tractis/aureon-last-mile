Distribution Scanning Flows

Two modes for different operational scenarios

Batch Mode (Primary — Efficient)

Batch Mode Flow
Step 1
View Pending
System groups packages
by andén using rules
→
Step 2
Select Andén
Pick target andén
to start a batch
→
Step 3
Scan Packages
Scan into batch
(validates belonging)
→
Step 4
Close Batch
Operator decides
when batch is ready
→
Step 5
Scan Andén PK
Confirm placement
→ status updated
EDGE CASES
  • Wrong zone package: Rejected — "Does not belong to this batch"
  • Unmapped comuna: Auto-routes to consolidation + flags for attention
  • Multiple batches: Operator creates several small batches per andén

Quick-Sort Mode (Fallback — Crowded Floor)

Quick-Sort Flow
Step 1
Scan Package
Scan any barcode
→
Step 2
See Destination
Big display:
"→ Andén 3" or "→ Consolidación"
→
Step 3
Scan Andén PK
Confirm placement
→ status updated
KEY DIFFERENCE
  • No pre-grouping — scan one package at a time
  • System creates a single-item batch automatically
  • Still requires andén PK scan for traceability
