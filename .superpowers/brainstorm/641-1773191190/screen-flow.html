Epic 4A: Pickup Verification — Screen Flow

4 screens for the core scanning workflow

→

Screen 1: Manifest List

/pickup — Shows all unconsumed manifests grouped by external_load_id. Each card shows retailer name, order count, package count. Two tabs: "Active" (default) and "Completed". Tapping an active card creates/resumes a manifest record and navigates to scanning.

→

Screen 2: Scanning

/pickup/scan/[external_load_id] — Auto-focused text input captures barcode scanner keystrokes. Shows progress bar (packages scanned / total), scan history list (last 10 scans with status icons), and live counters (verified, errors, duplicates). Distinct audio feedback for each result. Error popup for "not found".

→

Screen 3: Discrepancy Review

/pickup/review/[external_load_id] — Lists all missing/unscanned packages. Crew adds a note per missing package (required before signing). Also shows "not found" scans (barcodes scanned but not in manifest). "Continue Scanning" button to go back if needed.

→

Screen 4: Signature & Completion

/pickup/complete/[external_load_id] — Summary stats (verified, missing, precision, time elapsed). Operator signature pad (required). Optional client signature toggle. "Complete & Generate Receipt" button. Receipt shows full audit trail with discrepancy notes — the legal custody transfer document.

Key UX Decisions

Input method

Hardware scanner (BT/USB) → focused text input + Enter. No camera.

Feedback priority

Audio first (distinct tones), haptic second, visual third. Rapid scanning support.

Error handling

"Not found" → error sound + popup. Crew dismisses and continues. Logged for review.

Completion gate

If discrepancies exist, crew must add notes before signing. Notes appear on receipt.
